# Forget Password API

This API endpoint allows users to initiate the password reset process by requesting a password reset link.

## Endpoint

**POST** `/api/users/forgot-password`

## Payload Structure

Below is the structure of the JSON payload required for initiating the password reset process:

| **Field** | **Type** | **Description**                                     |
| --------- | -------- | --------------------------------------------------- |
| `email`   | `string` | The email address of the user requesting the reset. |

## Example Request Payload

```json copy filename="forgot-password.json"
{
  "email": "john.doe@example.com"
}
```

## Response Structure

The following table outlines the structure of the response after a successful password reset request:

| **Field** | **Type** | **Description**                                       |
| --------- | -------- | ----------------------------------------------------- |
| `message` | `string` | A confirmation message indicating the email was sent. |

## Example Response

```json
{
  "message": "If the email exists in our system, a password reset link has been sent."
}
```

## Argument Types

Here is a table summarizing the argument types for better clarity:

| **Argument** | **Type** | **Required** | **Description**                |
| ------------ | -------- | ------------ | ------------------------------ |
| `email`      | `string` | Yes          | The email address of the user. |

## Notes

- The email address provided must be associated with an existing account.
- For security reasons, the response message does not indicate whether the email exists in the system.
